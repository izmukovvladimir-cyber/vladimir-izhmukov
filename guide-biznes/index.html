Claude for Small Business: тридцать один готовый навык, которые снимают рутину с небольшой команды

Набор реально существует, называется Claude for Small Business, вышел 13 мая 2026 года, и навыков в нём тридцать один, вместо трёх из ролика. Главное, о чём почти никто не говорит: сами навыки лежат в открытом репозитории на GitHub под лицензией Apache 2.0, их можно скачать и прочитать бесплатно, без подписки и без единого подключённого сервиса. Внутри разобраны три сильнейших навыка до рабочего состояния, честная картина по доступам и отдельный файловый режим для тех, у кого Stripe и QuickBooks не работают.

Читается за 14 минут. Внутри: точное название набора и прямые ссылки на официальную страницу и репозиторий, установка двумя путями с копируемыми командами плюс третий путь без подписки, разбор навыков business-pulse, invoice-chase и job-post-builder до готового результата с указанием источников данных и формата вывода, честный раздел про то, какие данные куда уходят и что подключать не обязательно, чинилка на шесть типовых сбоев и рабочая замена Stripe и QuickBooks для России с тремя инженерными промптами, которые повторяют логику навыков на выгрузках из банка.

Начни с первого раздела, там точное название и ссылки, по которым всё проверяется за минуту. Дальше выбери свой путь установки из второго раздела: с подпиской идёшь через Cowork, с Claude Code через две команды в терминале, без того и другого просто забираешь тексты навыков с GitHub и читаешь их как готовые методички. Три разбора навыков читай под свою задачу, подряд их читать незачем. Если платишь и принимаешь оплату в России, седьмой раздел про замены важнее всех остальных, там лежит рабочая схема на выгрузках из банка без единого коннектора.

Что это на самом деле и где лежит

Сначала точные факты, потому что вокруг этого набора уже много пересказов с искажёнными названиями. Всё ниже проверяется по двум ссылкам за минуту.

  • Точное название: Claude for Small Business. Анонс вышел 13 мая 2026 года на официальном сайте Anthropic. Если ищешь по запросу вроде «код для малого бизнеса», не найдёшь ничего, название именно такое.
  • Что это по форме: плагин внутри среды Claude Cowork. Отдельного тарифа под него нет, доплаты сверх подписки тоже нет. Включается тумблером внутри Cowork.
  • Сколько там навыков: в открытом репозитории лежит тридцать один навык. В официальном анонсе фигурирует цифра пятнадцать, потому что там считают готовые сценарии верхнего уровня, а в репозитории видны все составные части.
  • Где лежат сами навыки: репозиторий anthropics/knowledge-work-plugins на GitHub, папка small-business, лицензия Apache 2.0. Это открытый исходник, каждый навык это текстовый файл SKILL.md с инструкцией на английском.
  • Что стоит денег: сама среда Cowork требует платный тариф, начиная примерно с 20 долларов в месяц. На бесплатном тарифе Cowork не открывается. Чтение навыков с GitHub при этом бесплатно и подписки не требует.
Самое ценное в этой истории это открытый репозиторий. Anthropic по сути выложила методички, по которым их модель делает финансовые сводки, письма должникам и наём. Эти тексты работают как инструкция для любой сильной модели, даже если у тебя нет ни Cowork, ни подписки, ни одного подключённого сервиса.

Три пути установки, выбери свой

Пути отличаются тем, что у тебя уже есть. Третий работает вообще без денег и без подключений, и именно он актуален для большинства читателей из России.

Путь первый: через Cowork, если есть платная подписка

  1. Открой claude.com/plugins и найди плагин small-business в каталоге.
  2. Установи его и включи тумблер внутри Cowork.
  3. Подключи сервисы, которыми реально пользуешься. Подключать все подряд не нужно, навыки пропускают то, чего нет, и помечают такие блоки как недоступные.
  4. Напиши в чат «get me started», плагин сам проведёт по остатку настройки.

Путь второй: через Claude Code, две команды

Промпт
claude plugin marketplace add anthropics/knowledge-work-plugins claude plugin install small-business@knowledge-work-plugins

Первая команда добавляет каталог плагинов Anthropic, вторая ставит именно набор для малого бизнеса. После установки навыки подхватываются автоматически, вызывать их отдельно не нужно, модель сама берёт нужный, когда ты формулируешь задачу словами вроде «кто мне должен» или «собери сводку по бизнесу».

Путь третий: забрать тексты навыков без подписки

Промпт
git clone https://github.com/anthropics/knowledge-work-plugins.git cd knowledge-work-plugins/small-business/skills ls

После этих трёх строк у тебя на диске лежат все тридцать один навык. Каждый это папка, внутри файл SKILL.md с полной инструкцией. Открывай любой блокнотом, копируй содержимое и вставляй в любую сильную модель как системную инструкцию. Работает в ChatGPT, в Claude на бесплатном тарифе, в Qwen. Без git можно проще: открой ссылку на репозиторий в браузере, зайди в нужную папку, нажми на SKILL.md и скопируй текст с экрана.

Названия трёх папок, которые нужны под задачи из ролика: business-pulse это утренняя сводка, invoice-chase это письма должникам, job-post-builder это сборка вакансии. Остальные двадцать восемь тоже стоят просмотра, там есть margin-analyzer по марже, price-check по ценам, handle-complaint по жалобам и tax-prep под налоговый период.

Навык первый: утренняя сводка по бизнесу

Папка business-pulse. Задача навыка собрать одну страницу, по которой видно состояние дел, вместо обхода шести вкладок по утрам.

Откуда он берёт данные

  • Деньги: QuickBooks по остатку на счетах, выручке и дебиторке. PayPal и Square по поступлениям и транзакциям.
  • Продажи: HubSpot по воронке и движению сделок.
  • Операционка: Google Calendar по встречам и дедлайнам, Gmail по срочным письмам, Slack или Teams по внутренним сигналам.
  • Поддержка: Intercom, Zendesk, Shopify, если они подключены.

Важная деталь механики: навык бьёт во все источники одновременно, вместо обхода по очереди. Поэтому сводка на шести подключённых сервисах собирается за то же время, что на двух. Если сервиса нет, блок просто выпадает из отчёта с пометкой, что данных нет, и ошибку это не вызывает.

Что получается на выходе

Одна страница, где каждый показатель помечен цветом состояния: зелёный, жёлтый, красный. Считается возраст дебиторки, покрытие воронки, динамика выручки к прошлому периоду. Самое сильное место в инструкции навыка это требование к формулировкам рисков. В самом тексте SKILL.md прямо записано, что фраза про «несколько просроченных счетов» бесполезна, а полезна строка вида «3400 долларов от компании Acme, просрочка 47 дней». Навык обязан называть конкретного клиента, конкретную сумму и конкретное число дней.

  • Числа идут впереди текста, каждый показатель с дельтой к прошлому периоду.
  • Разделы без данных помечаются как недоступные. Выдумывать числа навык не станет.
  • Если по факту ничего не изменилось, навык обязан написать это прямо, вместо того чтобы наполнять отчёт водой.
  • Срабатывает на формулировках вроде «как идут дела», «собери сводку», «что я упускаю», «введи меня в курс».
Поправка к тому, что чаще всего пересказывают про этот навык. Stripe в списке источников сводки официально не заявлен, там QuickBooks, PayPal, Square, HubSpot, почта, календарь и мессенджер. Stripe появляется в другом навыке, в письмах должникам, и там он подключается по желанию.

Навык второй: письма должникам

Папка invoice-chase, версия 0.2.0. Навык собирает список тех, кто не заплатил, и пишет письма под каждого. Здесь есть тонкость, которую обычно передают неверно.

Откуда берутся долги

  • QuickBooks: отчёт по возрасту дебиторки, в работу берутся счета с просрочкой от одного дня.
  • PayPal: закрытые транзакции за последние семь дней, фильтр по статусу проведения. Это нужно, чтобы не написать письмо тому, кто уже заплатил вчера.
  • Stripe: подключается по желанию, навык спрашивает разрешение на этапе настройки.

Как он выбирает тон письма

Навык раскладывает клиентов на три группы по истории платежей: платит вовремя, иногда задерживает, задерживает постоянно. Тон письма выбирается по группе, в которую попал конкретный клиент. То есть человеку, который просто забыл впервые за два года, уйдёт мягкое напоминание, а тому, кто тянет каждый раз, уйдёт жёсткая формулировка сразу.

Здесь важная поправка. В пересказах обычно звучит, что письма становятся жёстче по мере того, как человек молчит. В самой инструкции навыка такой логики нарастания по времени нет. Тон определяется историей платежей клиента внутри одного прогона. Нарастание жёсткости от письма к письму ты при желании настраиваешь сам, это не встроенное поведение.

Что получается на выходе

Сначала сводная таблица: клиент, сумма, сколько дней просрочки, выбранный тон, способ отправки. Под таблицей полные тексты писем. Ничего не уходит само: навык обязан дождаться твоего явного одобрения перед отправкой или постановкой письма в очередь. Это встроенный предохранитель, снять его нельзя.

Навык третий: сборка вакансии

Папка job-post-builder. Из всех трёх этот работает лучше всего вне подключений, потому что ему почти не нужны внешние сервисы.

Навык проходит шесть этапов и отдаёт три готовых документа. Сначала он собирает вводные по роли: название, обязанности, требования, город или удалёнка, вилка, из скольких этапов состоит собеседование. Дальше идёт в поиск и разбирает от трёх до пяти живых вакансий на такую же роль, чтобы текст опирался на живой рынок с реальными формулировками. Только после этого начинает писать.

Текст вакансии

Файл в формате Word. Пишется через результат работы. Сухой перечень обязанностей туда не идёт. Если навык нашёл на твоём диске или в Google Drive твой прошлый шаблон вакансии, он берёт за основу его и сохраняет твоё оформление.

Гид по собеседованию с оценочным листом

Разбит по этапам: телефонный скрининг, разговор с руководителем, разговор с будущими коллегами, практическое задание, финал. На каждом этапе свои вопросы, уточняющие подвопросы и шкала оценки с описанными опорами на балл 1, балл 3 и балл 5. Это и есть самая полезная часть, потому что она убирает ощущение «вроде норм» и переводит собеседование в числа.

Шаблон оффера

Документ с квадратными скобками под данные кандидата, вилку, условия и юридические оговорки. При желании навык умеет отправить его в DocuSign на подпись, но только черновиком и только после твоего подтверждения.

Границы навыка записаны в нём прямым текстом, и это честно. Он не публикует вакансию на площадках, это остаётся на тебе. Он не отбирает и не ранжирует откликнувшихся. Он не отправляет документы на подпись без твоего явного согласия. И в инструкции стоит отдельное требование показать оффер юристу перед использованием, потому что шаблон общий и под твою страну не адаптирован.
С чего начать в соцсетях

Доступы: что куда уходит и что можно не подключать

Раздел, который в восторженных пересказах обычно пропускают. Утренняя сводка работает только тогда, когда ты отдал ей доступ к почте, календарю и деньгам, и это стоит понимать до включения тумблера.

  • Куда уходят данные. Подключая коннектор, ты разрешаешь читать содержимое сервиса на серверах Anthropic во время работы навыка. Это не локальная обработка. Для писем и календаря это означает, что тексты писем и названия встреч уходят наружу.
  • Что подключать не обязательно. Обязательного нет вообще ничего. Каждый навык рассчитан на неполный набор источников и просто выкидывает из отчёта то, чего нет. Сводка на одном подключённом QuickBooks соберётся и будет полезной.
  • Минимально осмысленный набор под сводку: один источник денег плюс календарь. Почту и мессенджер добавляй последними, они дают наименьший прирост пользы и наибольший объём личных данных наружу.
  • Право на действие. Ни один из трёх разобранных навыков не совершает необратимых действий сам. Письмо должнику, оффер в DocuSign, любая операция с деньгами требуют твоего явного подтверждения. Это заложено в самой инструкции навыков, галочкой в настройках такое не снимается.
  • Отзыв доступа. Коннектор отключается там же, где подключался, и после отключения навык продолжает работать на оставшихся источниках. Разбирать всю сборку заново не нужно.
Практическое правило, если работаешь с чужими персональными данными клиентов. Не подключай почту целиком. Вместо этого выгружай нужный отчёт из бухгалтерии файлом и давай навыку только его. Файловый режим из следующего раздела закрывает ровно эту задачу и заодно снимает вопрос про доступы полностью.

Что из этого работает в России и чем заменить Stripe и QuickBooks

Честно: коннекторы из коробки для России бесполезны почти полностью. Полезной остаётся сама логика навыков, и она переносится на выгрузки файлами за один вечер.

  • Что не работает: Stripe и QuickBooks в России недоступны. PayPal и Square тоже отпадают. Сам Cowork требует платного тарифа и зарубежного способа оплаты, российская карта не пройдёт.
  • Чем заменить приём платежей: ЮKassa, Точка, Тинькофф Касса, Робокасса. Все они умеют выгружать реестр операций в CSV или Excel из личного кабинета.
  • Чем заменить бухгалтерию: 1С, Контур.Эльба, Моё дело. Отчёт по дебиторке с возрастом долга выгружается из любого из них.
  • Чем заменить HubSpot: amoCRM или Битрикс24, обе выгружают сделки списком.
  • Что работает без замен вообще: сборка вакансии. Этому навыку внешние сервисы почти не нужны, ему нужны твои вводные и поиск по рынку. Забирай текст навыка с GitHub и работай с ним в любой модели.

Файловый режим: как повторить утреннюю сводку без коннекторов

Схема простая и занимает пять минут в день. Раз в сутки выгружаешь три файла: реестр поступлений из банка или кассы, отчёт по дебиторке из бухгалтерии, список активных сделок из CRM. Складываешь в одну папку. Дальше вставляешь промпт ниже и прикладываешь файлы. На выходе получаешь ту же одностраничную сводку с цветами и названными рисками, ради которой всё и затевалось. Данные при этом уходят только те, которые ты сам приложил.

Промпт
Действуй как финансовый директор небольшой компании, который каждое утро кладёт собственнику на стол одну страницу. Ты не пересказываешь таблицы, ты вытаскиваешь из них решения. Числа у тебя идут впереди текста. Я прикладываю файлы: 1. Реестр поступлений за период: [название файла] 2. Отчёт по дебиторке с возрастом долга: [название файла] 3. Список активных сделок из CRM: [название файла] Мои вводные: Период сравнения: [например, эта неделя против прошлой] Нормальная отсрочка по договору: [например, 14 дней] Плановая выручка на месяц: [сумма] Обязательные платежи в ближайшие 14 дней: [сумма и на что] Собери сводку по правилам. 1. Деньги: остаток, поступления за период, дельта к прошлому периоду в рублях и процентах. 2. Дебиторка: общая сумма, разложенная по возрасту на группы до 30 дней, 30-60, 60-90, больше 90. 3. Воронка: сумма активных сделок, сколько из них сдвинулось за период, сколько стоит без движения дольше двух недель. 4. Риски: не больше пяти строк. Каждая строка обязана содержать конкретное название клиента, конкретную сумму и конкретное число дней. Формулировки вида «есть просроченные счета» запрещены, вместо них пиши «ООО Ромашка, 340 000 рублей, просрочка 47 дней». 5. Кассовый разрыв: сопоставь ожидаемые поступления с обязательными платежами на 14 дней вперёд и скажи прямо, хватает или нет. Формат ответа: одна страница. Сверху четыре строки показателей, у каждой пометка состояния зелёный, жёлтый или красный по твоей оценке. Ниже блок рисков. В самом низу три строки: что сделать сегодня, что на этой неделе, за чем просто наблюдать. Если в файлах нет данных под какой-то раздел, напиши по нему «нет данных» и не считай его. Не достраивай недостающие числа. Если по факту ничего существенно не изменилось, так и напиши одной строкой, вместо того чтобы наполнять отчёт словами.

Письма должникам на российских реалиях

Промпт
Действуй как руководитель отдела по работе с дебиторской задолженностью. Твоя цель вернуть деньги и сохранить рабочие отношения с теми, кто платит нормально. Ты подбираешь тон под историю конкретного клиента. Одинаковые письма всем подряд недопустимы. Прикладываю отчёт по дебиторке: [название файла] Прикладываю реестр поступлений за 7 дней: [название файла] Мои вводные: От кого письмо: [имя, должность, компания] Отсрочка по договору: [число дней] Неустойка по договору: [есть или нет, размер] С кем нельзя жёстко: [перечисли клиентов, которых нельзя терять] Сделай следующее. 1. Сначала вычти из списка долгов всех, кто есть в реестре поступлений за последние 7 дней. Писать тому, кто уже заплатил, недопустимо. 2. Разложи оставшихся на три группы: платит вовремя и задержал впервые, иногда задерживает, задерживает систематически. 3. Подбери тон под группу. Первым мягкое напоминание с допущением, что счёт просто потерялся. Вторым нейтральное деловое письмо с точной датой. Третьим жёсткое письмо со ссылкой на договор, суммой неустойки и следующим шагом. 4. Напиши каждое письмо целиком, на русском, готовым к отправке. Тема письма отдельной строкой. Обращение по имени. Сумма, номер счёта и число дней просрочки в первом абзаце. Формат ответа: сначала таблица со столбцами Клиент, Сумма, Дней просрочки, Группа, Тон. Под таблицей полные тексты писем по порядку. Ничего не считай отправленным. В конце добавь строку с вопросом, какие письма я одобряю к отправке. Если по клиенту не хватает данных для определения группы, отнеси его к нейтральной и пометь это явно.
Оба промпта повторяют логику оригинальных навыков, включая главное требование про конкретику в формулировках рисков и про обязательное подтверждение перед отправкой. Разница только в источнике данных: там коннекторы, здесь твои файлы. Для большинства небольших компаний в России второй вариант выходит и быстрее, и спокойнее по данным.

Чинилка: шесть типовых сбоев

Разобраны по принципу «что видишь, что на самом деле, что делать».

  • Плагин не находится в каталоге. Чаще всего человек ищет по названию из пересказа. Ищи ровно small-business, набор называется Claude for Small Business, других вариантов написания нет.
  • Команда установки отвечает, что маркетплейс не найден. Первая команда добавляет каталог и должна отработать до второй. Проверь, что выполнил именно две строки по порядку, и что в имени нет опечатки: anthropics с буквой s на конце.
  • Cowork не открывается совсем. На бесплатном тарифе его нет. Нужен платный тариф от Pro и выше, отдельной оплаты за сам набор при этом не берут.
  • Сводка приходит наполовину пустой. Это штатное поведение при неполном наборе подключений. Ошибки здесь нет. Навык помечает недостающие блоки и считает по тому, что есть. Хочешь полнее, добавь ещё один источник, начиная с денег.
  • Навык не запускается сам, когда просишь. Он срабатывает на смысл задачи. Формулируй обычными словами владельца бизнеса: «кто мне должен денег», «собери сводку по бизнесу», «нужна вакансия на роль». Абстрактные просьбы вроде «помоги с финансами» его не поднимают.
  • Письма должникам пришли с одинаковым тоном. Значит, навык не смог разложить клиентов по истории платежей, чаще всего из-за короткого периода в выгрузке. Дай отчёт с историей хотя бы за три месяца, на одном месяце группы не различаются.
Отдельно про версии. Навыки в репозитории живые, их правят. У навыка про должников на момент написания версия 0.2.0. Если поведение разошлось с описанным здесь, открой SKILL.md в репозитории и прочитай текущую версию, это один экран текста и он всегда точнее любого пересказа, включая этот.

С чего начать сегодня, чтобы не растянуть на месяц

Порядок, при котором первый полезный результат приходит в тот же вечер, независимо от того, есть у тебя подписка или нет.

  1. Открой репозиторий и прочитай business-pulse целиком. Это один экран текста на английском, десять минут с переводчиком. После него станет понятно, за счёт чего навык вообще полезен, и дальше ты сможешь править любой из них под себя.
  2. Выгрузи три файла за вчерашний день: поступления из банка, дебиторку из бухгалтерии, сделки из CRM. Если CRM нет, хватит первых двух.
  3. Прогони промпт утренней сводки на этих файлах. Первый результат будет сырым, это нормально. Посмотри, каких вводных ему не хватило, и допиши их в промпт под себя. Со второго прогона сводка становится рабочей.
  4. Прогони промпт по должникам. Здесь эффект виден сразу в деньгах, потому что почти у всех в дебиторке лежат забытые счета, до которых просто не доходили руки.
  5. Сохрани оба промпта себе с уже заполненными вводными про отсрочку, вилку и обязательные платежи. Дальше ежедневный прогон занимает пять минут, потому что меняются только приложенные файлы.
  6. Вакансию собирай тогда, когда она реально понадобится. Забери job-post-builder с GitHub заранее и положи рядом, чтобы не искать в момент, когда человек нужен вчера.
Через месяц вернись и сравни свои промпты с текущими версиями навыков в репозитории. Anthropic правит их по мере того, как собирает жалобы живых пользователей, и оттуда регулярно можно забрать пару чужих находок бесплатно.
